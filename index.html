Feaster

Over the past 8 months, I have been working with Feaster, a digital marketing platform that provides users with custom deals and offers on local food and drink on Facebook Messenger. Through Feaster I have had the opportunity to fully create and execute a visual identity, marketing materials, a landing page, and more for a project that I am very excited and honored to be an integral component of.

Please note that all materials here are copyright and property of Feaster, Inc. Please visit out landing page for more information.

Logo Iterations

My first task as a member of the Feaster team was to develop a logo to fuel all of our branding identity. My goal was to communicate to the user the key features that the bot delivers -- that is, an optimal way to receive personalized deals on local food and drink while saving money.

I started off by brainstorming with some rather cliche illustrations iconic of eating and savings, and trying to find a way to combine them.

Some of the brainstorming process behind the Feaster logo.

After some more brainstorming, I came up with the idea of a piggy bank. Pigs are iconic of eating, and piggy banks are iconic of saving. I decided to add a fork and spoon on the surface of the piggy bank to empahsize the use of this bank as a place for storing the savings a user would garner by using Feaster.

I am very proud to present the final product.

The final version of Feaster's logo

Logo Text Iterations

My next task was to decide on the way in which we wanted to present our logo text. I experimented with countless fonts and styles, but nothinq quite clikced. After endless searching to find the right font to convey the feel I wanted to, I ended up lettering the text on my tablet.

The final version of Feaster's logo.

WeatherOrNot

This project is a Rails webapp called WeatherOrNot, an applictaion that uses user feedback forms to create for users personalized suggestions about what to wear based on the weather for the current day. The inspiration behind the app is that, rather often, the same weather conditions feel quite different for different people. WeatherOrNot is a way for users to always know how to dress prepared without having to take time out of their morning rotinues.

Technologies: Ruby on Rails, Sass, Twitter Bootstrap

This past semester I took a course about Ruby on Rails. The course covered many of the integral features and powers of the "Rails magic," and I loved it so much that I have started working on developing some side projects on my own using the Rails framework. (In fact, I have gotten so used to firing up the Rails server that I initially developed this site in Rails and then converted it to pure HTML/CSS/JS to put up on Github pages.) Rails is the first backend framework that I have worked with, and it has deinitely inspired me to continue exploring backend development.

WeatherOrNot began as my final group project for the Rails course mentioned above. My project parteners were Anne Lin and Michelle Han. After the semester completed, I forked our project repo and continued to work on the app on my own. As of now, it is still a work in progres. A few of the features I have implemented and am in the process of implementing are:

  • AJAX calls to a Weather API in order to gather realtime weather stats (previously, I seeded the database with random weather instances. The user can select if they want to see real data or random data to play around with more diverse weather conditions.)
  • Integrated Bootstrap in order to add to the styling I previously implemented with vanilla CSS.
  • Improved the UI of the daily feedback forms.
  • Added a calendar which users can use to access and edit their feedback forms from previous days

Please be sure to check back soon to see the final version of the app once I put it up on Heroku!

Personal Site Iteration

I for some reason decided that it would be a good idea to build up a personal site from sratch -- no Bootstrap templates, no framerworks, just a blank HTML document with equally blank stylesheets and scripts. Although I am aware that all of the resources for developers out there renders this approach rather impratical, I am very glad that I did it. I've never been more confident in my understanding of HTML, CSS, and JavaScript, and it was a lot of fun and extremely rewarding to be able to StackOverflow my way through every obstacle that I encountered and end up with a product that is completely my own!

Note: for now, please fullscreen your browser, since I am in the process of optimizing the responsiveness of the homepage!

Technologies: HTML, CSS, JavaScript, Sass

During my first semester of college, I took a course called The Web Design Decal . It was in this class that I was first exposed to the basics of web development. I fell in love immediately. I have been wanting to code up a personal site ever since, and I am very happy to share the final result!

The Process

My ultimate goal with this site was to create something that fits with my style as both a designer and developer. This site isn't just a place to showcase my projects, but also one of my projects itself.

I started off by defining my color palette, settig up a layout, and choosing fonts. I also made some fun icons that pop up throughtout the site, adding an element of interactiveness to make the site more than just a static portfolio.

The layout I started out with was a very basic one. I had a standard navbar dividing the site up into a few different categories, and clicking on each link led to a different page. After implementing this layout, I got pretty bored of looking at it. I wanted instead something that was a bit more fluid, a bit more interactive, and a lot more fun. So I decide to reimplement the layout into a one page site

Currently, all of the content is "stored" either off to the side or at the bottom of the viewport. I implemented page transitions using a combination of CSS and jQuery animations. A menu is also "stored" to the left of the viewport, which a visitor to my site can also use to navigate from section to section.

Conclusion

I had a lot of fun implementing this site. I feel so much more confident in not only my web devleopment abilities, but also my problem-solving abilities in both coding and learning in general.

I'm happy to say that I was able to create a product that feels very "me," and I'm now even more exited to keep learning and developing.
